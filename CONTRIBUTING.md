# 🤝 Contributing to the SaaS Command Center

First off, thank you for considering contributing. This repository is built on the philosophy of **Speed over Perfection** and **Brutal Simplicity**. If you have a high-leverage strategy that moves the needle (ARR, retention, acquisition), we want it in the master playbook.

## 🚀 The Brainiac Standard

We don't accept fluff. We don't do "nice to haves". 

If you are submitting a new playbook or updating an existing one, it must adhere to the **Vibe-Coding Command Center** ethos:
- **Action-Oriented:** Tell the founder exactly what to do. No theoretical essays.
- **AI-Ready:** Include explicit, copy-pasteable prompts that an AI agent (Cursor, Gemini, Claude) can use to execute the steps.
- **Measurable:** Define clear, empirical success metrics. How do we know this worked?

## 🛠️ How to Contribute

1. **Fork the Repository:** Create your own copy of the `SaaS-Blueprint`.
2. **Create a Branch:** `git checkout -b feature/your-awesome-playbook`
3. **Add or Edit a Playbook:** 
   - Navigate to the appropriate folder inside `Dummy Data/` (e.g., `Dummy Data/Acquisition/TikTok Ads/`).
   - Create or edit the `PLAYBOOK.md` file.
4. **Follow the Template:** Ensure your `PLAYBOOK.md` follows the standard structure (see below).
5. **Commit & Push:** Commit your changes with a clear, concise message.
6. **Open a Pull Request:** Submit your PR against our `main` branch. Briefly explain what your playbook solves and why it's high-leverage.

## 📝 The `PLAYBOOK.md` Template

Every playbook **MUST** follow this exact format:

```markdown
# 🧠 PLAYBOOK: [Strategy Name]

## 🎯 Objective
[One sentence. What is the absolute minimum we are trying to achieve?]

## 🚀 The Brainiac Vibe
[The mindset required. High-leverage, direct, and results-oriented. Why are we doing this?]

## 🛠️ Action Steps
1. **[Step 1]:** [Specific action]
2. **[Step 2]:** [Specific action]
3. **[Step 3]:** [Specific action]

## 🤖 Vibe Coding Prompts
*Copy-paste these into your AI to get started instantly.*

> "Prompt 1: Execute step 1..."

> "Prompt 2: Review and refactor..."

## 📈 Success Metrics
- **[Metric 1]:** [How do we know we succeeded?]
- **[Metric 2]:** [What number needs to go up?]
```

## ⚠️ Guardrails
- **No API Keys:** Never commit `.env` files or hardcode secrets into your playbooks.
- **No Self-Promotion:** Do not use playbooks merely to link to your paid courses or tools unless it is objectively the best tool for the job and stated clearly.
